# 问题描述

在「需求」「商业」场景页（`pages/requirement/index`、`pages/business/index`）中，归档话题条目来自收件箱 `kind=session` 的过滤结果，且时间线把归档块整体拼在**当前会话消息之上**。收件箱中归档会话按**最近活跃降序**，导致**最新归档**出现在列表**最顶**，当前对话消息被挤到下方，与聊天产品「底部为当前对话、向上翻看更早记录」的习惯相反。归档数量多时缺少分页，一次拉全量收件箱也不利于扩展。

**复现**：商业页存在多条已归档话题时，最新归档提示条显示在屏幕最上方，当前输入区上方应为当前话题消息而非最新归档。

# 根因分析

1. 前端 `computed.timelineItems` 使用 `[...archivedTopicTips, ...messages]`，且 `archivedTopicTips` 顺序与收件箱归档段一致（新在前），视觉上最新归档占据顶部。
2. 无按智能体维度的归档分页接口，无法实现「触顶加载更旧归档」。
