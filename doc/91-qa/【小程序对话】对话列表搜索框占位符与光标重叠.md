# 问题描述

对话列表页顶部搜索框在获得焦点、出现输入光标时，**占位提示「搜索会话或智能体」仍显示**，与光标重叠，不符合常见输入框体验（聚焦后占位符应隐藏）。

复现：微信开发者工具或真机打开「对话」页，点击搜索框即可见。

# 根因分析

微信小程序端部分场景下原生 `<input>` 的 `placeholder` 在聚焦时仍渲染，与光标叠在同一行首，属平台表现与预期不一致。通过在**聚焦时动态清空 `placeholder`、失焦后恢复**，与常见 IM/搜索列表交互一致。
