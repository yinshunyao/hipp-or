# 问题描述

在 `hipp-admin` 中将「智能客服管理」菜单挂载到「系统配置」父菜单下后，点击菜单页面空白，前端控制台报错：

- `[Vue Router warn]: Record with path "/system/agentManager" is either missing a "component(s)" or "children" property.`

# 关联文档

| 类型 | 文档路径 |
|:---|:---|
| 原始需求 | `hipp-or/doc/01-or/【智能客服管理】Dify智能体接入管理.md` |
| 开发设计 | 待定位 |
| 测试设计 | 待定位 |

# 根因分析

1. 本项目后台菜单走动态路由构建，`/auth/getMenuList` 返回的菜单项中若 `component` 为空、前缀不规范（如带前导 `/`），或组件路径写法不兼容，会导致动态路由节点未解析出 `component`。
2. 该场景下路由记录 `path=/system/agentManager` 在前端构建后出现「无 `component` 且无有效 `children`」的非法路由，触发 Vue Router 警告并导致页面空白。

# 涉及文件

- `hipp-or/hipp-admin/src/utils/routerHelper.ts`
  - 增加动态路由组件解析兼容：支持前导 `/`、`@/`、`src/`、带扩展名写法。
  - 增加智能客服管理路由的兜底映射（`agentManager`、`/system/agentManager`）。
  - 对无 `component` 且无子路由的非法菜单节点进行跳过，避免注册无效路由。

# 设计约束更新

- 动态菜单路由需要保证「叶子路由必须可解析为有效页面组件」。
- 对历史菜单数据中的路径写法差异需前端做兼容解析，避免因配置细节差异导致线上空白页。
