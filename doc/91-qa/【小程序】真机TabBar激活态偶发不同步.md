# 问题描述

真机环境下，点击底部某个 Tab 后页面能够正确 `switchTab`，但底部导航激活态偶发不更新，表现为旧 Tab 仍保持高亮或当前 Tab 未高亮。该问题在模拟器中当前不稳定复现。

# 根因分析

`custom-tab-bar/index.js` 仅依赖 `getCurrentPages()` 读取当前栈顶路由做激活态同步。  
在部分真机上，`switchTab` 后路由事件先触发、页面栈后更新，导致短时间内持续读到旧栈顶页面，`activePath` 被回写为旧值（例如切到“商业”后仍高亮“需求”）。

# 涉及文件

- `hipp-or/hipp-uni/custom-tab-bar/index.js`
  - 在组件 `attached` 时注册 `wx.onAppRoute`，路由变化时优先用事件路径直接校准 `activePath`。
  - 将激活态重试窗口扩展为 `0/80/180/320/500/800/1200ms`，覆盖慢机型页面栈延迟。
  - `onChange` 时先更新本地 `activePath` 提供即时反馈；`switchTab` 完成/失败后再执行路由兜底同步。
- `hipp-or/doc/02-dr/04 小程序对话/小程序导航与我的页设计.md`
  - 补充“TabBar 激活态真机同步稳定性优化”设计说明与约束。
- `hipp-or/doc/03-tr/04 小程序对话/小程序导航与我的页测试设计.md`
  - 新增真机回归用例 `TC-N29`，覆盖连续切换场景下的激活态一致性验证。

# 设计约束更新

- 真机 `switchTab` 激活态同步不能依赖单次生命周期回调，必须包含延迟重试校准。
- Tab 激活态的最终值以当前页真实路由为准；点击事件赋值仅用于即时视觉反馈。
- 路由校准优先级应为「路由事件路径 > 页面栈路径」，页面栈只作为兜底来源。
