# 【小程序对话】键盘弹起顶栏与菜单消失

## 问题描述

在微信小程序中，对话输入框聚焦弹出软键盘时，整页被上推过多，顶部自定义导航区域与系统状态/胶囊区域被顶出可视范围，表现为「上方内容超出页面、菜单不见」。

真机上还出现键盘直接遮住输入框、输入内容不可见的情况。

## 根因分析

1. `textarea` 默认 `adjust-position=true` 时，微信会整体上推页面以保证光标可见；与全高 flex 布局及固定 `uni-nav-bar` 叠加时，容易出现过度上推，将顶栏移出视口。
2. 初版修复采用 `padding-bottom` 加在容器上，但 Tab/非 Tab 分支逻辑导致场景页（需求/商业）的 `keyboardBottomPadStyle` 实际返回 `{}`，真机键盘直接遮住输入框。

## 当前方案

统一「键盘高度占位 view」方案，去掉 Tab/非 Tab 分支：

- `MP-WEIXIN` 关闭 `adjust-position`。
- 通过 `uni.onKeyboardHeightChange`（全局）+ textarea `@keyboardheightchange`（组件级）双通道获取键盘高度。
- 在 `input-bar` 下方、flex 容器末尾插入 `.kb-spacer`（`height = keyboardHeight`）。
- flex 布局自动让 `msg-list`（`flex:1`）收缩、`input-bar` 上移到键盘上方，顶栏不动。
- `input-bar--kb-up` 类在键盘弹起时去除 tabbar / safe-area 的多余底边距。
- 键盘收起时 `keyboardHeight` 归零、spacer 消失、布局恢复原位。

## 涉及文件

- `hipp-or/hipp-uni/common/mixins/mp-keyboard-offset.js`（重写）
- `hipp-or/hipp-uni/pages/requirement/index.vue`、`pages/business/index.vue`、`subpkg/chat/chat.vue`：统一接入 mixin，模板增加 `kb-spacer` 占位 view 与 `input-bar--kb-up` class。

无（交互仍为底部输入、上方消息列表；仅键盘避让实现方式从「整页上推」改为「底部占位 flex 收缩」）。
