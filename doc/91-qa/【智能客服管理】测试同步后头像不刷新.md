# 问题描述

在「智能客服管理」页面，当 Dify 平台已更换智能体头像与信息后：

- 在管理后台点击「测试」
- 提示信息更新成功（名称/描述等可刷新）
- **但头像未更新**（仍显示旧头像）

复现前提：Dify 侧头像内容有变更，但 `icon_url` 可能保持同一 URL（或同一路径）。

# 根因分析

管理后台头像图片使用 `icon_url` 作为 `<img src="...">` 的 URL。

当 Dify 更新头像但 `icon_url` 不变时，浏览器/中间缓存可能复用旧响应，导致即便点击「测试」并同步成功，前端仍渲染同一个 `src`，头像不触发重新拉取。

# 涉及文件

- `hipp-or/hipp-admin/src/views/Vadmin/AgentManager/AgentManager.vue`
  - 列表头像 `img src` 增加 cache-bust 参数（基于 `update_datetime`）
- `hipp-or/hipp-admin/src/views/Vadmin/AgentManager/components/Write.vue`
  - 弹窗信息区头像 `img :src` 增加 cache-bust 参数（基于 `update_datetime`，并在测试成功后兜底递增 nonce）
- `hipp-or/test/e2e/agent-manager/agent-manager.spec.ts`
  - 新增/调整用例：模拟「测试后列表刷新」场景，断言头像 `src` 的版本参数发生变化

# 设计约束更新

- 头像（`icon_url`）展示需考虑缓存：当远端资源同 URL 内容变化时，前端应通过版本参数或强制刷新策略确保用户可见更新。
