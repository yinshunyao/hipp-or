# 问题描述

管理端在「智能客服管理」中修改已接入智能体的展示信息（如名称、头像等）后，小程序侧**对话页**导航栏标题及收件箱中对应会话行仍显示修改前的名称；头像在收件箱依赖 `agent` 展示对象时亦可能因列表优先使用会话 `title` 与旧缓存感知而不一致。

复现：创建会话后返回管理端修改智能体名称/头像，小程序保持在对话页或返回收件箱再进入，仍看到旧名称。

# 根因分析

1. **标题**：`vadmin_chat_session.title` 在创建时写入智能体名，且可能与 `agent_name_snapshot` 长期一致；对话页与收件箱列表优先展示 `session.title`，未在「未改标题」场景下合并**当前**智能体名称。
2. **刷新时机**：对话页仅在 `onLoad` 拉取会话详情，从其他页返回同一聊天页时若不重新请求，无法看到后台刚改的配置；已补充 `onShow` 元数据刷新（仅标题与状态，不整表重载消息）。
