# 问题描述

微信小程序中，作为默认落地页的「需求」场景对话（`pages/requirement/index`）顶部原生导航栏区域**左上角返回/系统按钮不可见或不可点**，用户误以为返回按钮丢失。

# 关联文档

| 类型 | 文档路径 |
|:---|:---|
| 原始需求 | `hipp-or/doc/01-or/小程序整体设计.md` |
| 开发设计 | `hipp-or/doc/02-dr/04 小程序对话/小程序用户对话设计.md` |
| 测试设计 | `hipp-or/doc/03-tr/04 小程序对话/小程序用户对话测试设计.md` |

# 根因分析

1. **布局**：场景对话页根容器使用 `height: 100vh`。在微信小程序中，`100vh` 常与页面可视区域计算不一致，易导致根节点纵向占满**包含导航栏对应区域**，使内容与导航栏叠层，**遮挡**左上角返回箭头或胶囊触控区。
2. **底栏 API**：启用 `tabBar.custom` 后仍全局调用 `uni.setTabBarStyle`。该接口针对**原生 TabBar**，对自定义 TabBar 无效；个别基础库下可能与导航表现产生干扰，故在微信端不再调用。

# 涉及文件

- `hipp-or/hipp-uni/pages/requirement/index.vue`：`page { height: 100% }`，根容器改为 `height: 100%` + `min-height: 0`。
- `hipp-or/hipp-uni/pages/business/index.vue`、`subpkg/chat/chat.vue`、`pages/chat/index.vue`：同类布局修正，避免同类遮挡。
- `hipp-or/hipp-uni/common/mixins/theme.js`：`setTabBarStyle` 仅非 `MP-WEIXIN` 执行。

# 设计约束更新

- 主包/分包内全屏纵向布局的页面，优先使用 `page { height: 100% }` + 根节点 `height: 100%` / `min-height: 0`，避免根级 `100vh` 与微信原生导航栏叠层。

# 说明（产品行为）

- 微信对 **Tab 根页**在**页面栈仅一层**时，原生**可能不展示「返回」**（无上一页可回），属平台规则；若需回到「对话」等其它 Tab，应使用底部 **TabBar** 切换。
