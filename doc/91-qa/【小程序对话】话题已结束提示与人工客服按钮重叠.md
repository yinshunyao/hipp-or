# 问题描述

会话详情 `subpkg/chat/chat` 在话题已结束且智能体仍上架时，底部同时展示「本话题已结束…」提示（Vant `NoticeBar`）与「联系人工客服」按钮。按钮使用**固定定位**叠在提示条区域右侧，与多行文案同一视觉层，导致**提示后半句被遮挡**，用户无法读全说明。

复现：微信开发者工具或真机进入已结束话题的会话页，观察底部提示与客服按钮。

# 根因分析

`联系人工客服` 容器 `.human-fab` 使用 `position: fixed; right; bottom`，叠在处于文档流中的 `readonly-notice-slot`（`NoticeBar`）之上，未为提示区预留右侧避让空间，小屏或多行文案时发生遮挡。

# 涉及文件

- `hipp-or/hipp-uni/subpkg/chat/chat.vue`：将提示条与客服按钮包入纵向容器 `.topic-end-stack`，客服按钮改为流内布局（`align-self: flex-end`），去掉 `fixed` 叠层。
- 后续：去掉 `.topic-end-stack` 与输入栏重复的 `safe-area-inset-bottom`（仅输入栏保留安全区）；将 `$mp-scene-detail-list-pad-bottom` 从 120rpx 降为 24rpx，避免提示区与输入框之间大块空白；`scroll-view` 列表增加 `min-height: 0` 以利 flex 收缩。
