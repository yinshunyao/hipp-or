

# 问题范例
## Q1 英语教育平台
```text
# 每行一句话
我想做一个英语教育平台，包含管理端和教学端，管理端只需要web平台，教学端是进行教学时老师和学生使用，需要支持web/苹果和安卓的平板
管理端需要支持下属机构接入，对学生、老师、课程等进行管理，支持老师的薪酬管理，替换我现有的管理平台，现有的教学是基于腾讯会议。
教学端不需要白板，课件比较简单，英语阅读，生词本和完型填空等，管理端会有按照不同等级课程和课时数来计算。
替代我现有的管理系统（薪酬管理做的不好）， 现在教学直播是基于腾讯会议。目标用户主要是机构，老师和学生。
自己旗下的机构内部使用，主要是作为公司内部成本中心，下属机构是直营分校，但是需要各自有账号登录管理。
我现在已经有一套管理系统了，只是薪酬管理功能无法使用，现在这个系统薪酬管理完全不支持，老师薪酬是课程级别和课时来定的。
现在已经有不少老师和1000名学员基于腾讯会议进行直播授课。
关于直播和录课，我希望直接对接第三方平台，不用自己开发。
```

## Q2 需求分析、人力和预算评估
```text
我想做一个小程序
面向哪些有软件需求，但对技术不太了解的用户
提供需求分析、人力和预算评估功能，需要用户输入他的软件需求
我暂时没有想到变现模式

# 追问
你觉得这类用户在使用你的小程序时，最让他们头疼的具体问题是什么？是怕被外包公司坑，还是完全不知道自己的需求该怎么描述清楚？
# 回答
对，怕被外包公司或者个人坑，不知道合理的预算以及风险；
另外，可能自身对业务需求考虑得比较清楚，技术相关引入的成本没有考虑。

# 追问
关于解决方案的具体形式，你是打算让用户输入文字描述后由系统自动生成评估报告，还是希望通过问答互动的方式一步步引导他们理清需求呢？
# 回答
第一个版本，我只想通过问答方式简单实现

# 追问
虽然你暂时没想好变现模式，但你觉得未来谁会最愿意为这个‘避坑指南’付费？是这些有需求的个人创业者，还是可能有一些想获客的外包公司愿意买单？或者你有其他初步的想法吗？
# 回答
我想两方面，第一个是避坑指南 付费； 第二是我自身也是 独立开发者，希望能帮我引流
```
