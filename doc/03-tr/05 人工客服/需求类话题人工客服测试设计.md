# 测试范围

- 人工会话创建/复用、`system` 归档提示内容、收件箱 `kind=session` / `kind=staff`、参与者鉴权、人工消息同步发送、流式接口对人工会话的拒绝。
- 小程序：悬浮入口、人工会话消息展示（含 `system`）、客服端列表与发送。

# 测试用例（摘要）

| 前置条件 | 操作步骤 | 预期结果 | 自动化 |
|:---|:---|:---|:---|
| 存在已归档 `dify` 会话 | POST `/human-support/sessions` | 返回 `session_kind=human_support`，含 `assigned_human_*`，首条消息含 `system` | 需集成环境 |
| 同上会话再次发起 | POST 同上 | 返回同一会话 id（复用） | 需集成环境 |
| 已有人工会话进行中，另选**不同**已归档会话发起 | POST `/human-support/sessions`（新 `source_session_id`） | **不新建**会话 id，向原会话追加 `system`；客服仍为原分配人 | 需集成环境 |
| 人工会话 | POST `.../messages/stream` | HTTP 422 | 需集成环境 |
| 无参 | `format_archive_topic_context` 单测 | 文本含标题、会话 id、用户/智能体标签 | pytest 已覆盖 |
| 已存在人工会话，且合并了多个归档上下文提示 | 打开人工会话消息列表 | 每条归档提示条展示为“标题”且**不含** `归档话题:` 前缀；标题具备下划线等可点击态；点击可进入对应归档会话 | 手工（小程序/工作台） |
