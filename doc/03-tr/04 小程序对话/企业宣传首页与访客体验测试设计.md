# 测试范围

- 小程序：首页展示与 Tab（含页脚备案号展示）、首页跳转需求/商业、首页右侧「需求咨询」悬浮气泡可见性与跳转、无 token 时自动游客登录、体验用尽提示与登录引导、「我的」页游客态。
- 后端：`POST /auth/mp/guest`、场景解析扩展字段、`create_session`/发消息在游客**单场景**第三次归档后的拒绝（业务码 46004）；需求与商业配额独立。

# 测试用例（手工 / 标识自动化）

| 编号 | 前置条件 | 操作步骤 | 预期结果 | 自动化 |
|:---|:---|:---|:---|:---|
| TC-H-01 | 未安装或清缓存 | 打开小程序 | 默认进入首页 Tab，展示企业宣传区块 | 否 |
| TC-H-01B | 在首页 | 滚动至页面最下方 | 页脚信息区展示备案号文案 `备案号：蜀ICP备2026013590号-2X` | 否 |
| TC-H-02 | 无正式登录 | 点击首页「需求咨询」 | 进入需求页并可加载场景智能体 | 否 |
| TC-H-02B | 在首页 | 上下滚动长页 | 右侧「需求咨询」气泡始终固定在视口右侧可见（不随内容滚走） | 否 |
| TC-H-02C | 在首页 | 点击右侧悬浮「需求咨询」气泡 | 与 TC-H-02 一致，进入需求 Tab | 否 |
| TC-H-03 | 游客身份 | 仅在「需求」场景完成 2 次话题归档（`is_topic_closed=true`） | 该场景第三次新建会话或发消息被拒绝（46004）；「商业」场景独立计数，仍可体验至本场景 2 次归档用尽 | 否 |
| TC-H-03B | 游客身份 | 仅在「商业」场景完成 2 次话题归档 | 该场景用尽后拒绝；「需求」若未用尽仍可继续使用 | 否 |
| TC-H-04 | 游客 | 分别调用 `scene-agent?scene=requirement` 与 `scene=business` | 各自返回的 `guest_scene_archived_count` 仅统计本场景类型下的已归档数 | 否 |
| TC-H-05 | 正式登录用户 | 同上归档多次 | 不受 2 次游客限制（与现网一致） | 否 |
| TC-H-06 | 已绑定手机/微信的正式用户，本地已执行退出登录（无 token） | 在首页点击「需求咨询」或「商业咨询」 | 无法通过 `/auth/mp/guest` 静默换票；路由守卫跳转登录页；「我的」不展示为已登录正式用户 | 否 |
| TC-H-07 | 游客身份（`mp_guest`，「我的」页展示游客提示） | 点击「手机号登录」 | 进入 `pages/login/login` 短信登录页，**不**被重定向回首页 | 否 |
