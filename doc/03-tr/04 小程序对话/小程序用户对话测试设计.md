# 测试范围

- hipp-api `apps/mp_chat`：收件箱查询（`GET /mp/chat/inbox` 支持 `kind=session` 与合并模式）、创建会话、发送消息（Mock Dify）、会话更新与删除、权限（未登录 401）。表设计为逻辑 ID + 无 DB 外键，关联由 JOIN/应用层校验；用例不依赖数据库级级联。
- hipp-uni：关键流程以手工或后续 H5+Playwright 补测；本阶段以接口自动化为主。
- 底部菜单：`对话` Tab 为**归档话题**列表（无智能体入口行）；`需求` / `商业` 为进行中对话入口；小程序默认进入 `需求` 页（见 `permission.js`）。

# 自动化说明

- **框架**：`pytest` + `unittest.mock` / `httpx` MockTransport（与 `test_agent_manager` 风格一致）。
- **E2E**：小程序真机/开发者工具以手工为主；若编译 H5，可后续在 `hipp-or/test/e2e` 增加页面级用例。
