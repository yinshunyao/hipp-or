# 测试范围

- hipp-api `apps/mp_chat`：收件箱查询、创建会话、发送消息（Mock Dify）、会话更新与删除、权限（未登录 401）。表设计为逻辑 ID + 无 DB 外键，关联由 JOIN/应用层校验；用例不依赖数据库级级联。
- hipp-uni：关键流程以手工或后续 H5+Playwright 补测；本阶段以接口自动化为主。

|:---|:---|:---|:---|:---|
| TC01 | Mock Dify 返回 answer 与 conversation_id | 调用发送消息 API（blocking） | 会话写入 assistant 消息，session.dify_conversation_id 更新 | pytest + httpx mock |
| TC02 | 无 | 对 `send_message` 中 Dify 非 2xx | 抛出/返回可识别错误，不伪造成功 | pytest |
| TC03 | 已存在会话 | GET inbox | 包含 kind=session 行 | 可选集成测试 |
| TC04 | 无 | `DifyStreamAccumulator` 解析 `message` 增量 | `full_answer` 与事件顺序一致 | pytest 单元测试 |
| TC05 | 无 | 对话列表页点击搜索框获得焦点 | 占位提示不与光标重叠（聚焦时无占位或等价表现） | 手工（微信开发者工具/真机） |
| TC06 | 已登录进入 `pages/chat/chat` | 聚焦输入框、输入多行文字 | 文本换行，输入区增高至上限后可滚动；键盘弹出时输入区不被遮挡 | 手工（真机优先） |
| TC07 | 同上 | 查看发送区 | 为圆形图标按钮（向上箭头），无「发送」文案；无内容时按钮不可用 | 手工 |
| TC08 | iPhone 等有底部横条机型 | 查看输入栏底部 | 与 Home Indicator 有间距，不重叠 | 手工 |
| TC09 | 已登录进入 `pages/chat/chat`，网络正常 | 输入较长中文并点击发送 | 消息区**立即**出现右侧用户气泡，且**可见完整正文**（非仅色块）；随后助手气泡流式或整段出现 | 手工（微信开发者工具/真机） |
| TC10 | 同上 | 发送一条消息直至完成 | 用户消息在过程中可标为发送中/结束后为已发送；无异常「无字气泡」 | 手工 |
| TC11 | 模拟助手接口失败（断网或非 2xx） | 发送一条消息 | 用户气泡正文仍可见；助手侧为失败提示；用户消息展示**失败**状态且可点击**重试**，重试后再次请求 | 手工 |
| TC12 | 用户存在历史会话，关联智能体被下架（status 非 published） | 进入收件箱与会话页 | 可看到该会话和历史消息；会话标识为已下架；输入与发送禁用 | API 自动化 + 手工 |
| TC13 | 用户存在历史会话，关联智能体被物理删除 | 进入收件箱与会话页 | 可看到该会话与历史消息（使用快照名称/头像或默认占位）；输入与发送禁用 | API 自动化 + 手工 |
| TC14 | 会话对应智能体已下架或删除 | 调用 `POST /mp/chat/sessions/{id}/messages` 与 `/messages/stream` | 返回明确业务错误（如 `agent_unavailable`），且不调用 Dify | pytest + mock |
| TC15 | Dify 流式返回首行为 `结束状态 True/False`（含 `:` / `：` 变体） | 调用流式发送并消费 SSE | 客户端收到的实时 `answer` 增量不包含结束状态行，仅保留正文 | pytest 单元测试 |
| TC16 | 已创建会话且智能体可发送 | 调用 `POST /mp/chat/sessions/{id}/messages/stream` 完成一轮对话后退出并重新进入会话 | `GET /sessions/{id}/messages` 能查询到本轮 user + assistant 两条历史，且 `last_message_preview` 已更新 | API 自动化 + 手工 |
| TC16 | 已登录进入 `pages/chat/chat`，助手返回 Markdown（标题、列表、代码、链接） | 发送问题并等待助手回复 | 助手气泡按 Markdown 样式展示，不显示原始 `#`/`**`/``` 等源码标记；`http/https` 链接可点击，其它协议不作为可跳转链接 | 手工（微信开发者工具/真机） |
| TC17 | 已登录进入 `pages/chat/chat`，存在助手长回复 | 查看助手消息气泡与屏幕左右边 | 助手消息区横向铺满可视宽度（与列表左右内边距解耦），右侧无大块空白；用户消息仍为右侧限宽气泡 | 手工（微信开发者工具/真机） |
| TC18 | 用户已登录 | 打开小程序或从登录页登录成功 | 默认进入 `pages/chat/index`，不进入 `pages/index` | 手工 |
| TC19 | 小程序进入任意页，观察 TabBar | 检查底部导航项与图标 | 仅显示「对话 / 工作台 / 我的」，无「首页」；对话使用消息语义图标 | 手工 |

# 自动化说明

- **框架**：`pytest` + `unittest.mock` / `httpx` MockTransport（与 `test_agent_manager` 风格一致）。
- **E2E**：小程序真机/开发者工具以手工为主；若编译 H5，可后续在 `hipp-or/test/e2e` 增加页面级用例。
