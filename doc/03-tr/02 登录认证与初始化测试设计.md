# 测试范围

- 账号密码登录：手机号/账号登录标识识别与失败提示兼容性（`method=0`）。
- 短信验证码登录（`method=1`，`platform=1`）：发码频控、验证码校验、**手机号不存在时自动建号**、已存在用户直接登录。
- 短信登录在 `platform=0` 的行为（若实现为禁止自动建号）：不存在用户应失败且不得建号。
- 用户字段：自动创建用户的 `user_type`、`telephone`、默认昵称/`name`、随机密码哈希、`is_staff`/`is_active`。
- 小程序前端：默认短信登录 UI、验证码倒计时、切换账号密码、密码显隐；**无微信登录入口**；登录态与 `userType` 持久化。
- 启动初始化：`run.sh` 在数据库不存在时自动建库后继续缺表检查与初始化。

# 自动化测试标识

- 后端框架：**pytest**，目录：`hipp-or/hipp-api/test/`。
- 建议新增/调整用例文件：覆盖 `platform=1` + `method=1` 自动建号、错误验证码不建号、已存在用户复用（可置于 `test_apps/test_auth/` 下专用模块）。
- 历史微信登录用例（`/auth/wx/login`）：若接口保留，可降为非小程序主路径的回归子集；小程序主路径以短信用例为准。
- 前端：当前 `hipp-uni` 未接入 Playwright，小程序端以手工回归为主；后续补齐 e2e 后再迁移为自动化。
