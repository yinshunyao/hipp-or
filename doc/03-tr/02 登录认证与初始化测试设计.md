# 测试范围

- 账号密码登录：手机号/账号登录标识识别与失败提示兼容性。
- 微信授权登录：用户资料 + 手机号授权链路、自动建号与重复登录复用。
- 用户类型字段：`user_type`、`wx_nickname`、`wx_avatar` 的写入、更新与返回。
- 小程序前端：密码显隐、微信授权按钮、登录态与 userType 持久化。
- 启动初始化：`run.sh` 在数据库不存在时自动建库后继续缺表检查与初始化。

# 测试用例

| 编号 | 前置条件 | 操作步骤 | 预期结果 | 自动化 |
|:---|:---|:---|:---|:---|
| TC-LOGIN-01 | 存在系统用户 | 使用账号密码登录 `method=0` | 登录成功，返回 `user_type=system` | pytest |
| TC-LOGIN-02 | 无 | 输入非法登录标识登录 | 返回格式错误文案，不查库 | pytest |
| TC-WX-01 | 无该手机号用户 | 调用 `/auth/wx/login` 并传 `code+nickname+avatar` | 自动创建用户，`user_type=wechat`，返回 token | pytest |
| TC-WX-02 | 已有该手机号用户 | 再次微信登录并携带新昵称头像 | 不新建用户，更新 `wx_nickname/wx_avatar` | pytest |
| TC-WX-03 | 微信 code 无效 | 调用微信登录 | 返回 `无效Code` | pytest |
| TC-WX-CONF-01 | `.env` 已配置 `WECHAT_APPID/WECHAT_KEY` | 调用微信登录链路 | 服务端优先使用环境变量配置，不依赖系统设置表 | pytest |
| TC-FE-01 | 小程序登录页 | 切换密码显隐后输入密码 | 输入值不丢失，展示状态切换正确 | 手工回归 |
| TC-FE-02 | 小程序登录页 | 微信授权后完成登录 | 前端保存 `token`、`refreshToken`、`userType` | 手工回归 |
| TC-INIT-DB-01 | MySQL 实例可连接，目标数据库不存在 | 执行 `hipp-api/run.sh`，观察初始化输出与后续启动 | 脚本自动创建数据库，随后按核心表检查决定是否执行 `init`，最终进入 `main.py run` | shell+pytest（可选） |
| TC-INIT-DB-02 | MySQL 用户无建库权限，目标数据库不存在 | 执行 `hipp-api/run.sh` | 脚本打印“自动创建数据库失败”并退出，退出码非 0 | shell |
