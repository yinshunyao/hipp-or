# 背景
我想开发一个微信小程序，需要符合微信小程序UI规范

# 整体布局
下方是菜单，分别是话题、需求、商业 和 我的
- 话题 是 归档话题 的列表，
- 需求 和 商业，实际上是 智能客服 的对话页面
- 我的 就是一般的 个人信息页面

# 话题
1. 标题是 话题
2. 最上方是 搜索框，提示语 是 "搜索话题或者记录"
3. 下方类似于聊天记录表，每个 话题 展示 图标、标题、简要描述 和 时间
4. 长按 在下方 会弹出 菜单， 自上而下分别是 置顶、继续对话、修改标题、删除会话和取消
5. 每个 话题 点击可进入详情页面，类似聊天窗口，从上往下依次是
- 标题是 该话题名称， 左侧有 "返回" 按钮，图标展示，不要文字
- 聊天页面， 展示对话内容
- 提示，内容是：本话题已结束，无法继续发送。请返回列表新建对话，或在会话列表长按该会话选择「继续对话」。
- 输入框，发送按钮（都无效）

# 需求 和 商业
1. 标题是 需求 或者 商业， 左侧有 "返回" 按钮，图标展示，不要文字
2. 聊天页面， 展示对话内容
- 按照时间线，最上方一般是 已经归档话题： "[消息提示] XXXX"
- 正常对话内容
- 输入框， 发送按钮

# 我的
1. 头像，一个好看的banner
2. 编辑资料
3. 应用设置
4. 关于我们

点击一般就是资料显示，或者资料修改页面

# 注意
1. 话题列表需要出两种图，有弹窗和没有弹窗的
2. 下面菜单上边缘，不要有圆角，不好看
3. 弹框，聊天内容和卡片的圆角弧度都小一些，不要太大
