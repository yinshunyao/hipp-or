# 背景
我想开发一个微信小程序，需要符合微信小程序UI规范

# 目标
1. 提供符合微信小程序 UI 规范的整体界面与信息架构。
2. 支持「话题 / 需求 / 商业 / 我的」四个一级入口的核心页面展示与交互。

# 要求
## 功能性要求
### 整体布局
- 下方是菜单，分别是 **话题、需求、商业 和 我的**。
- **话题**：归档话题列表。
- **需求**、**商业**：智能客服对话页面（两者仅标题与入口不同，交互一致）。
- **我的**：个人信息入口页。

### Tabbar（底部菜单）
- **必须使用自定义 Tabbar**，不再使用 uniapp 的 `pages` 配置模式来实现底部 tabbar。
- 下面菜单上边缘 **不要有圆角**（直边更符合期望观感）。

### 话题
1. 页面标题是「话题」。
2. 最上方是搜索框，提示语是「搜索话题或者记录」。
3. 下方类似聊天记录表：每个话题展示 **图标、标题、简要描述、时间**。
4. 长按话题弹出菜单，自上而下分别是：**置顶、继续对话、修改标题、删除会话、取消**。
5. 点击话题进入详情页（类似聊天窗口），从上往下依次是：
   - 顶部：标题为该话题名称；左侧为「返回」按钮（**仅图标，不要文字**）。
   - 聊天页面：展示对话内容。
   - 提示：本话题已结束，无法继续发送。请返回列表新建对话，或在会话列表长按该会话选择「继续对话」。
   - 输入框与发送按钮（在归档话题详情中 **均无效**）。

### 需求 / 商业
1. 页面标题分别是「需求」或「商业」；左侧为「返回」按钮（**仅图标，不要文字**）。
2. 聊天页面展示对话内容：
   - 按时间线从上到下；最上方一般是已归档话题提示：「[消息提示] XXXX」。
   - 正常对话内容。
   - 输入框与发送按钮（有效）。

### 我的
1. 头像 + 一个好看的 banner。
2. 编辑资料。
3. 应用设置。
4. 关于我们。
5. 修改密码页面需符合常见安全操作：必须输入当前密码、新密码、确认密码后才可提交。
6. 我的相关子页面（至少包含「修改密码」「关于我们」）顶部导航栏在深浅模式下保持主题一致，不得固定浅色。

（点击进入资料显示页或资料修改页，具体页面形态可在后续设计中确定）

## 非功能性要求（视觉与交互）
- 弹框、聊天内容、卡片的圆角弧度都 **小一些**，不要太大。
- 话题列表需要出 **两种图**：有弹窗和没有弹窗的。
