# 背景

1. 小程序用户与智能体对话必须由服务端统一鉴权、读取已上架智能体配置并调用 Dify，才能在**不暴露密钥**的前提下实现用户数据隔离、会话可追溯与运营侧上下架策略生效。若由客户端直连 Dify，无法满足上述约束。
2. 本文件界定 **hipp-api** 侧与用户智能对话相关的会话持久化、消息存储、向 Dify 的转发方式及流式交付；与小程序 UI、收件箱等行为的分工以 `hipp-or/doc/01-or/【小程序对话】用户智能对话.md` 及开发设计为准。
3. 上游已定义平台对 Dify 智能体的接入与上架（`hipp-or/doc/01-or/【智能客服管理】Dify智能体接入管理.md`）。对接 Dify Chat 应用的请求/响应形态可参考 `hipp-or/doc/01-or/Dify API对接参考.md` 及同目录下的 API 参考材料（如 `.mhtml`），以实现与联调一致。

# 目标

1. 持久化用户消息与助手回复及会话元数据，使小程序能够按会话分页查询历史并在列表中展示最近摘要。
2. 使用平台存储的智能体配置调用 Dify 官方 Chat 接口（`POST {api_server}/v1/chat-messages`），以 **streaming** 为主路径将模型输出交付给客户端；流正常结束后将助手内容落库并更新会话上的 `dify_conversation_id` 等字段。
3. 保持与 Dify 的多轮连续：首次发送不携带 `conversation_id`，根据 Dify 响应保存并复用 `conversation_id`；后续同一会话请求必须携带已保存的 `dify_conversation_id`。
4. 对助手回复正文做约定处理：若内容最前存在「结束状态 True」或「结束状态 False」行（表示需求沟通完成状态），**滤除该行**后再作为对用户可见的助手内容存储与展示，避免状态标识混入对话正文。
5. 保留历史会话可读性：当会话关联智能体已下架或已删除时，允许用户继续查询该智能体信息（历史快照）与消息记录，但禁止发起新的发送请求。

# 要求

## 功能性要求

- **会话与消息范围**：须落库的内容至少包括用户发送的文本、经 Dify 返回并处理后的助手文本；会话须能保存 Dify 返回的 `conversation_id`（字段名与表结构见开发设计）。收件箱合并、会话 CRUD、分页拉取消息等接口清单以 `hipp-or/doc/02-dr/04 小程序对话/小程序用户对话设计.md` 为准。
- **智能体与密钥**：仅允许使用管理端已上架（`published` 且未软删除）的智能体记录中的 `api_server`、`app_key` 等配置；**禁止**由客户端传入 Dify 服务地址或密钥。
- **历史会话与智能体状态**：会话详情和消息查询接口必须支持返回「已下架/已删除」智能体的历史信息（至少包括名称、头像占位、状态标识），用于客户端历史展示；当源智能体被物理删除导致主表无记录时，展示字段由会话侧冗余快照或约定默认值兜底。
- **Dify 侧用户标识**：`user` 等标识须与开发设计一致（例如 `user-{平台用户 id}`），保证不同登录用户隔离。
- **发送路径**：须提供流式发送接口（如 `POST .../messages/stream`）；可与非流式（blocking）接口并存作为降级，具体路径与行为以开发设计为准。
- **发送拦截**：发送接口在写入用户消息或调用 Dify 之前，必须校验会话关联智能体状态；若智能体非「已上架且未删除」，返回明确业务错误（例如“智能体已下架/删除，暂不支持继续对话”），并且不得继续转发到 Dify。
- **错误可机读**：发送拦截命中时，除展示文案外需返回稳定业务错误码（当前约定 `46001`），用于客户端进入只读状态与提示。
- **附件**：当前版本**不支持**消息附件的上传与向 Dify 转发。

## 非功能性要求

- **流式优先**：主业务路径必须按 **streaming** 处理 Dify 响应（如 `response_mode=streaming`、`text/event-stream`），服务端向客户端的交付方式（如 SSE 透传）以实现设计与测试设计为准；不得仅依赖 blocking 作为唯一主路径而影响实时体验。
- **错误与一致性**：Dify HTTP 非 2xx、无有效 `answer` 或流异常中断时的错误返回、用户消息/助手消息落库顺序与失败场景以开发设计为准；须避免未定义的半截状态（例如流未正常结束却写入完整助手内容等）。
- **数据与查询**：表间关联不依赖数据库级外键；多表查询在 SQL/服务层显式 JOIN，与项目数据访问约定一致。
