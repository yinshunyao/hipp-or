# 背景

企业使用 Dify 平台搭建了多个智能客服（智能体），每个智能体通过 Dify 提供的 API 对外服务。每个智能体由一个 API 服务器地址和一个 APP_KEY 唯一标识。

当前缺乏统一的管理入口，运营人员无法在管理平台中对智能客服进行集中注册、连通性验证、信息查看和上下架管控，需要直接操作 Dify 后台完成管理工作，效率低且缺乏生命周期管理手段。

# 目标

1. 提供智能客服注册与编辑功能，支持运营人员通过 API 服务器地址和 APP_KEY 接入 Dify 智能体
2. 提供智能客服连通性测试功能，确保接入配置正确后才允许上架
3. 提供智能客服列表展示与多条件查询功能，方便运营人员快速定位和管理
4. 支持智能客服上架/下架生命周期管理，控制对外可见性

# 要求

## 功能性要求

### 智能客服添加/编辑

- 配置项包括：API 服务器地址、APP_KEY、智能客服类型、备注
- 智能客服类型为下拉选择，且支持手动输入新增类型
  - 暂时内置两个类型：需求分析、商业评估
- 配置页面支持「测试」「保存」「上架」三个操作按钮
- 只有测试通过后才允许上架
- 编辑功能可复用添加页面

### 智能客服测试

- 调用 Dify「应用基本信息」接口，获取名称、描述、标签、模式等信息：

```shell
curl -X GET '{API服务器}/v1/info' \
-H 'Authorization: Bearer {APP_KEY}'
```

返回示例：

```json
{
  "name": "My App",
  "description": "This is my app.",
  "tags": ["tag1", "tag2"],
  "mode": "advanced-chat",
  "author_name": "Dify"
}
```

- 调用 Dify「应用 WebApp 设置」接口，获取图标、主题色等展示配置，原文 JSON 存入数据库 `webapp_config` 字段：

```shell
curl -X GET '{API服务器}/v1/site' \
-H 'Authorization: Bearer {APP_KEY}'
```

返回示例：

```json
{
  "title": "My App",
  "chat_color_theme": "#ff4a4a",
  "chat_color_theme_inverted": false,
  "icon_type": "emoji",
  "icon": "😄",
  "icon_background": "#FFEAD5",
  "icon_url": null,
  "description": "This is my app.",
  "copyright": "all rights reserved",
  "privacy_policy": "",
  "custom_disclaimer": "All generated by AI",
  "default_language": "en-US",
  "show_workflow_steps": false,
  "use_icon_as_answer_icon": false
}
```

- 测试成功后在按钮下方展示智能客服信息：头像、名称、描述、标签、备注（本平台）、上架状态（草稿/上架）
- **新增**：测试成功并获取到智能体信息后，支持运营人员在管理端**修改并保存生效**以下「智能体原生信息」：
  - 头像（支持上传图片替换，参见管理端系统设置中上传图片的交互与校验方式）
  - 名称（name）
  - 描述（description）
  - 标签（tags）
  - 保存成功后，列表与详情展示以保存后的信息为准；再次点击「测试」将同步 Dify 最新信息（如需以 Dify 为准可重新测试后再编辑保存）

### 智能客服列表展示

- 展示字段：头像、名称、描述、标签、智能客服类型、备注（本平台）、上架状态（草稿/上架）
- 头像展示策略：当 `icon_url` 有值时**优先展示** `icon_url` 对应图片；否则展示 `icon` + `icon_background`
- 每行操作按钮：编辑、上架/下架切换、测试（同步最新信息）、删除（标记删除，数据库保留记录）

### 智能客服列表查询

- 支持名称、描述、标签的模糊查询，输入时实时更新结果
- 支持上架状态下拉筛选（默认全部），选择后实时更新结果
- 支持智能客服类型下拉筛选（默认全部），选择后实时更新结果

## 非功能性要求

- 前端样式与管理平台现有页面保持一致，表格和按钮风格统一
- 删除采用标记删除方式，数据库记录保留，页面不再展示
