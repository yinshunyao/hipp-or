# 背景

用户在与智能体完成一次「需求类」对话后，会生成对应的归档话题，用户可在对话页面中查看该归档话题内容。部分用户在阅读归档话题时仍会产生理解与落地疑问，需要引入人工客服进行解读与答疑，形成「智能体归档 → 人工补充解读」的闭环。分配到的**人工客服**需在自身工作台侧进入对应会话，能查看完整会话记录并持续回复，与用户侧小程序对话体验衔接。

除管理后台工作台外，人工客服还需能通过**小程序**登录，在**对话页面**中直接看到用户发来的消息并回复，与用户在同一套对话产品形态下完成接待，避免仅依赖 PC 端导致响应不及时。

# 目标

1. 在归档的「需求类」对话页面提供明确入口，用户可一键发起人工客服对话。
2. 当用户发起人工客服对话时，系统能自动分配 1 名可用人工客服并建立会话。
3. 在人工客服对话中，用户与客服双方都能看到触发本次咨询的归档话题上下文，便于连续沟通。
4. 人工客服在其**对话页（工作台）**内可查看当前会话、浏览完整会话记录，并可持续发送消息完成跟进。
5. 人工客服通过**小程序**登录后，在对话页面可看到用户发送的消息并回复，与用户侧消息互通。

# 要求

## 功能性要求

1. **入口展示**
   - 在「归档的需求类对话页面」提供人工客服入口。
   - 入口需在用户滚动查看对话内容时保持易于触达（例如悬浮入口或等效交互）。
2. **发起对话**
   - 用户点击人工客服入口后，系统创建（或复用）与人工客服的对话会话，并跳转进入该会话。
   - **同一用户与同一人工客服侧在「进行中」的咨询应合并为一条人工会话**：若已存在未结束的人工会话，从**其他**归档话题再次发起时，应**并入该会话**（追加归档上下文提示），**不得**再创建第二条人工会话或再次分配客服。
   - 跳转后页面标题需展示被分配人工客服的姓名（或可识别名称）。
3. **人工客服账号与角色**
   - 人工客服为管理员在「用户管理」中创建的账号。
   - 人工客服账号的角色名称固定为：`人工客服`。
4. **客服分配规则**
   - **首次**建立人工会话时随机选择 1 名人工客服即可（无需实现复杂排班/负载均衡）；并入已有进行中会话时**不重新分配**。
   - 分配动作由后端触发并返回分配结果（前端不做本地随机）。
5. **上下文透传（归档话题）**
   - 在人工客服对话会话中，必须展示「触发对话的归档话题」内容，展示形态为消息提示格式（与对话消息区分）。
   - 该提示消息需在会话中可见（至少在会话开始阶段可见），用于告知双方本次咨询来源。
   - **归档话题标签文案（合并模式）**：当同一人工会话合并来自多个归档话题的上下文提示时，每条归档提示使用“标签/提示条”样式展示**标题即可**，去除 `归档话题:` / `归档话题：` 前缀文案；标题需有**下划线**等链接态视觉，明确“可点击查看该归档话题”。
6. **对话页面形态**
   - 人工客服对话页面的交互与样式可复用现有对话页面能力（如需求/商业等对话页面的基础能力），但必须满足本 OR 中的展示与上下文要求。
7. **人工客服侧：对话页与会话记录（工作台）**
   - 人工客服账号登录后，在**对话页**（工作台中的单会话界面）须能展示当前人工客服会话，并与用户侧会话一一对应。
   - 提供**查看会话记录**能力：可查看与该会话相关的完整历史消息（含用户与人工客服双方消息；触发来源的归档话题提示按上文「上下文透传」展示），支持按通用对话习惯滚动浏览；若需独立「记录」入口（如弹层、侧栏），以可一次看清完整沟通为准。
   - 支持在会话内**新发起/发送消息**（回复用户），与即时对话产品一致：发送后消息进入会话记录，用户侧可收到（具体通道与小程序侧约定见 `hipp-or/doc/01-or/【小程序对话】用户智能对话.md`）。
   - 仅被分配或有权处理该会话的人工客服可查看与回复；权限与数据范围在开发设计中细化。
8. **人工客服侧：小程序对话页**
   - 具备「人工客服」角色的账号可**登录小程序**，进入与业务约定一致的**对话页面**（会话列表、进入会话、消息区等能力在开发设计中细化，须与现有小程序对话体验一致）。
   - 在对应会话中可**看到用户发送过来的消息**（按时间顺序展示，含与本次会话相关的历史消息范围在开发设计中明确）。
   - 可在会话内**发送回复**；用户侧可收到，即时性与展示规则与同文档其他对话相关约束、以及 `【小程序对话】用户智能对话.md` 中的约定对齐。
   - 若同一会话同时支持工作台与小程序端接待，须避免漏读、重复回复或两端状态严重不一致（具体策略在开发设计中细化）。

## 非功能性要求

1. **可用性**
   - 入口需避免遮挡核心内容且不影响对话页面基础操作（滚动、复制、发送等）。
2. **一致性**
   - 归档话题在原对话页面与人工客服对话页面中的提示消息格式保持一致（同一类信息同一视觉语义）。
3. **可追溯性**
   - 人工客服会话需能追溯到触发它的归档话题（至少通过展示的提示消息实现用户可见追溯）。
4. **人工客服侧一致性**
   - 人工客服在**工作台**与**小程序**对话页中的消息顺序、发送状态与失败重试等体验宜与平台内通用对话页一致，避免用户与客服两端语义错位（具体在开发设计中选型）。
