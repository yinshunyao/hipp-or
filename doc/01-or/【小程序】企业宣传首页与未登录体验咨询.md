# 背景

小程序需强化企业品牌展示，并降低用户使用「需求分析」「商业评估」智能体前的门槛：用户希望先了解服务能力，再在不注册账号的情况下有限次体验咨询；体验用尽后引导正式登录以继续使用完整能力。

参考站点视觉与信息结构见 workspace 内 `爱宝喜宝/luo/index.html`（爱宝喜宝软件工作室宣传页）。

# 目标

1. 将小程序首屏升级为与上述参考一致风格与信息层级的**企业宣传首页**，作为小程序对外形象入口。
2. 从首页可跳转至「需求」「商业」场景进行**咨询体验**：未登录（无手机号账号）用户在**需求分析**场景下最多完成 **2 次**话题归档、在**商业评估**场景下最多完成 **2 次**话题归档（每次以对话「话题结束」归档为计，两类场景**分别计数、互不占额度**）；任一类场景用尽后，该场景须登录方可继续使用，另一类未用尽的仍可体验。
3. 体验规则在后端强校验，前端提供明确提示与登录引导。

# 要求

## 功能性要求

- 首页：展示企业定位、技术优势、案例与联系方式等区块，视觉与版式参考 `爱宝喜宝/luo/index.html`，适配小程序（滚动、安全区、无外链脚本依赖）。
- 首页：在主页最下方（页脚信息区）增加备案号展示文案：`备案号：蜀ICP备2026013590号-2X`。
- 首页：右侧常驻展示「需求咨询」悬浮气泡（不随内容滚动消失）；用户点击后 `switchTab` 至「需求」页 `pages/requirement/index`，行为与首页底部 CTA「需求咨询」一致。
- TabBar 增加「首页」入口，企业首页为小程序默认打开页之一（与现有导航协调）。
- 未绑定手机号的访客：通过微信 `code` 换取后端颁发的**游客身份**（与正式账号区分），可调用现有对话接口；在「需求分析」场景下已归档话题数达到 **2**、或「商业评估」场景下已归档话题数达到 **2**（按各自 `service_type` 统计，`is_topic_closed=true` 视为一次归档）后，**在该场景**禁止新建会话与继续发送，并提示登录；另一场景独立计数。
- 「话题」收件箱、其他非上述两类智能体会话：以现有权限与产品策略为准；本需求聚焦需求/商业两场景的访客体验上限。
- 正式登录后使用原有账号体系，游客体验次数不计入本需求的「登录用户」限制（若后续需合并数据，另立需求）。

## 非功能性要求

- 游客身份标识与次数校验须在服务端实现，不可仅依赖前端本地计数。
- 文案与交互符合微信小程序规范；错误码与提示可测试、可追溯。
