# 功能设计

## 数据模型（`vadmin_chat_session` / `vadmin_chat_message`）

- **`session_kind`**：`dify`（默认，既有数据迁移后同值）| `human_support`。
- **`agent_id`**：人工会话为 `NULL`；智能体会话仍为智能体 id。
- **`assigned_human_user_id`**：人工会话必填，指向 `vadmin_auth_user.id`。
- **`source_archive_session_id`**：人工会话必填，指向触发咨询的已归档智能体会话 id。
- **消息 `role`**：在原有 `user` / `assistant` 上增加 **`system`**，用于「归档话题参考」提示（与双方普通消息区分）。

表级不增加数据库 FOREIGN KEY，关联在应用层维护（与既有 mp_chat 约定一致）。

## 角色与分配

- 管理员在「用户管理」中为账号绑定角色，**角色名称**须为 **`人工客服`**（与 OR 一致）。
- 发起人工会话时由**后端**在具备该角色且 `is_active` 的用户中 **`ORDER BY RAND() LIMIT 1`** 随机分配；若无可用客服则报错提示。

## API（前缀 `/mp/chat`，鉴权同现有 `AllUserAuth`）

| 方法 | 路径 | 说明 |
|:---|:---|:---|
| POST | `/human-support/sessions` | body：`{ source_session_id }`。校验：当前用户拥有该会话、且为 `dify`、`is_topic_closed=true`。**复用**：若该用户已存在**未结束**（`is_topic_closed=false`）的人工会话，则**不新建会话、不重新分配客服**，仅向该会话追加一条 `role=system` 的归档上下文消息；若该归档来源已在同一会话中出现过（会话行 `source_archive_session_id` 或历史 `system` 内容含 `来源会话 ID：{id}`），则直接返回该会话详情。否则新建会话并随机分配客服，插入首条 `system` 消息。 |
| GET | `/inbox?kind=staff` | 当前用户为 `assigned_human_user_id` 的人工会话列表（客服端收件箱）。 |
| GET | `/inbox?kind=session` | 在原有「仅已归档智能体会话」基础上，**增加** `session_kind=human_support` 会话（进行中的与用户的对话）。 |
| GET | `/sessions/{id}` | 会话详情：客户侧 `display_title` 为客服姓名；客服侧为「用户 xxx」。增加 `session_kind`、`viewer_context`（`owner` \| `assigned_staff`）。 |
| GET | `/sessions/{id}/messages` | 参与者：会话**拥有者**或**被分配的客服**。 |
| POST | `/sessions/{id}/messages` | 智能体会话：行为不变。人工会话：拥有者发 `user`，被分配客服发 `assistant`；不落 Dify。 |
| POST | `/sessions/{id}/messages/stream` | 仅智能体会话；人工会话返回 422，引导使用同步接口。 |

删除、改标题、置顶等仍仅**会话拥有者（C 端用户）**可操作（`assert_session_owner_customer`）。

## 归档上下文（`system` 消息）

- 文案由 `format_archive_topic_context` 生成，包含归档标题、来源会话 id、按时间顺序摘录的用户/智能体轮次。
- 小程序与 Web 工作台展示时与对话气泡区分（提示条/弱背景样式）。
- **合并模式的提示条文案与可点击态**：当同一人工会话合并多个归档上下文提示时，提示条主标题展示**归档话题标题**本身，去除 `归档话题:` / `归档话题：` 前缀；标题增加下划线（或等效链接态样式）以提示可点击；点击后打开对应来源归档会话详情（`source_archive_session_id` 或从文案解析 `来源会话 ID`）。

## 管理后台

- 在 `asyncRouterMap` 增加 **人工客服 → 客服工作台**，页面调用与小程序相同的 `/mp/chat` 接口（`kind=staff`、消息列表、同步发送）。

## 小程序

- **归档智能体对话页**（`subpkg/chat/chat`）：在「只读归档且智能体仍上架」时展示悬浮 **联系人工客服**，调用 `POST /human-support/sessions` 后 `redirectTo` 进入人工会话。
- **对话 Tab**：列表展示人工会话（标签「人工」）；人工会话不标「已结束」。
- **客服端收件箱（小程序）**：角色含 **`人工客服`** 时，底部 Tab「**对话**」页自动使用 `GET /mp/chat/inbox?kind=staff` 展示分配给自己的会话（导航栏标题为「客服接待」）；与 C 端「对话」同页不同数据。原「我的 → 客服接待」入口已移除；子包 `subpkg/chat/staff-inbox` 仅作 `switchTab` 到「对话」的兼容跳转。

## 边界与多端一致

- 客服在 **Web 工作台** 与 **小程序** 使用同一套会话 id 与消息接口；发送均为同步接口，避免流式与人工场景混用。
- 仅被分配客服或会话拥有者可拉取对应会话与消息。
