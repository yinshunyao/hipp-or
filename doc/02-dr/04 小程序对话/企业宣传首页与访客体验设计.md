# 功能设计

## 小程序端

### 页面与导航

- 新增主包页面 `pages/home/index`：单页滚动，区块包括顶部品牌区、首屏 Hero、技术优势、案例展示、联系我们与页脚信息；**页脚信息区**需展示备案号文案 `备案号：蜀ICP备2026013590号-2X`；配色与层次对齐 `爱宝喜宝/luo/index.html`（深灰蓝 Hero、`#2980b9` 强调色、卡片与分割线）。
- `pages.json`：将 `pages/home/index` 置于 `pages` 列表靠前位置；`meta.loginAuth: false`；`tabBar` 首项为「首页」，其后为话题、需求、商业、我的。
- `custom-tab-bar`：增加首页项与图标。

### 访客登录（无手机号）

- 微信小程序启动后，在访问需登录路由前，若本地无 token：调用 `wx.login` 获取 `code`，请求 `POST /auth/mp/guest`，成功后写入与普通登录一致的 `Authorization` token（及 refresh_token）。
- 路由守卫（`permission.js`）：无 token 时先尝试 `EnsureGuestToken`，再跳转登录页。
- 已持有游客 JWT（`user_type=mp_guest`）时，用户主动进入登录页（例如「我的」页「手机号登录」）**须放行**，不得仅因本地已有 token 将登录页 `replace` 回首页。
- 用户信息：`GetInfo` 兼容 `roles` 为空；根据 `user_type === 'mp_guest'` 在「我的」页展示「登录/注册」为主操作，弱化需正式账号的菜单（或跳转登录）。

### 首页跳转咨询

- 首页 CTA：「需求咨询」`switchTab` 至 `pages/requirement/index`；「商业咨询」至 `pages/business/index`。
- **需求咨询悬浮气泡**：在 `pages/home/index` 中，于 `scroll-view` **外侧**增加 `position: fixed` 的右侧悬浮入口（z-index 高于正文），文案「需求咨询」；垂直位置避开 TabBar 与安全区（与首页底部留白策略一致）。用户滚动长页时气泡相对屏幕固定、始终可见；点击与 CTA「需求咨询」相同，调用 `uni.switchTab({ url: '/pages/requirement/index' })`。
- 场景接口 `GET /mp/chat/scene-agent` 响应扩展字段：`guest_scene_archived_count`（**当前场景** `service_type` 下已归档话题数）、`guest_scene_trial_limit`（固定 2）、`guest_need_login`（游客且**本场景**归档数达上限且无进行中会话时为 true）。需求与商业**分别计数**。前端在需求/商业页展示顶部提示条，并在 `guest_need_login` 为 true 时引导登录。

### 错误处理

- 后端对超限请求返回业务码 `46004`（与现有 `46001` 等并列），消息为「免费体验次数已用完，请登录后继续使用」。前端在创建会话、发送消息失败时可用 `uni.showModal` 跳转登录页。

## 后端

### 游客用户

- 新增接口 `POST /auth/mp/guest`，Body：`{ "code": "<wx.login>" }`。
- 使用现有 `WXOAuth.get_code2session` 获取 `openid`；按 `wx_server_openid` 查找用户：
  - **不存在**：创建 `VadminUser`：`user_type='mp_guest'`，`is_staff=True`（与现有小程序用户信息接口兼容），`telephone` 为合法占位 11 位手机号（如 `199` + 8 位数字哈希），`password` 随机哈希，姓名「游客」，写入 `wx_server_openid`。
  - **已存在且 `user_type='mp_guest'`**：复用该用户并签发 token。
  - **已存在且为非 `mp_guest`（已绑定正式/微信账号）**：**拒绝**签发 token（返回业务错误，文案如「此微信已绑定账号，请先登录」），防止用户在前端执行「退出登录」后，仅因再次 `wx.login` 又拿到与原账号等效的 JWT，表现为登录态被静默恢复。
- 成功时返回与普通登录一致的 `access_token` / `refresh_token` / `user_type`。

### 体验次数

- 常量：`GUEST_SCENE_TRIAL_ARCHIVES_MAX = 2`（**每个** `service_type` 各 2 次，非两场景合计）。
- 统计：用户维度下，按智能体 `service_type` **分别**统计：`vadmin_chat_session` 中 `session_kind='dify'`、`is_topic_closed=true`，且关联智能体 `service_type` 等于当前场景（「需求分析」或「商业评估」）的会话数量。
- `create_session`：若用户为 `mp_guest` 且**当前智能体所属 service_type** 的上述计数 ≥ 2，拒绝并返回 `46004`。
- 发送消息（同步、SSE、WebSocket）：在会话为上述两类智能体时，若 `mp_guest` 且**该会话智能体 service_type** 的计数 ≥ 2，拒绝并返回 `46004`（防止异常残留会话）。
- `resolve_scene_agent`：返回扩展字段供前端展示；当游客且**本场景**计数 ≥ 2 且无进行中会话时，`guest_need_login=true`。
- 人工客服会话创建：游客直接拒绝或提示登录（本需求采用拒绝并提示）。

### 数据模型

- 不新增表；复用 `user_type` 与现有会话/智能体字段。
