# 功能设计

## 底部导航（TabBar）

- **变更前**：`对话` | `工作台` | `我的`。
- **变更后**：`对话` | `我的`（两栏）。
- **实现**：在 `hipp-uni/pages.json` 中从 `pages` 主包列表移除 `pages/work/index`；从 `tabBar.list` 移除工作台项。不再保留工作台 Tab 占位。

## 底部导航（自定义 TabBar）

- **目标**：不再使用 uni-app 默认 `pages.json -> tabBar.iconPath/selectedIconPath` 样式与图标渲染。
- **实现方案**：
  1. 在 `hipp-uni/pages.json` 设置 `tabBar.custom: true`，保留四个 Tab 页路径：`pages/chat/index`、`pages/requirement/index`、`pages/business/index`、`pages/mine/index`。
  2. 新增 `hipp-uni/custom-tab-bar/index.vue` 作为底部导航容器，使用自定义布局（固定底部、无圆角上边缘）与图标文案。
  3. Tab 点击通过 `uni.switchTab` 跳转，激活态由当前路由同步。
  4. `pages/chat/index`、`pages/requirement/index`、`pages/business/index`、`pages/mine/index` 补齐底部安全间距，避免内容被自定义 TabBar 遮挡。
- **约束**：不使用 Vant `van-tabbar` 组件，避免微信小程序端出现自定义栏挂载不稳定导致底栏不可见的问题。

## 会话列表头像与 Tab 激活态视觉优化（本次变更）

- **头像底色柔化**：
  1. 在 `pages/chat/index.vue` 中将头像背景改为通过函数计算的柔化色（支持 `#rgb` / `#rrggbb` / `rgb(a)`），统一降到低透明度，降低高饱和底色突兀感。
  2. 当服务端未提供头像底色时，继续使用主题渐变底色作为兜底，保持深浅色模式一致。
  3. 头像容器增加细边框，提升与列表背景的层次区分。
- **Tab 激活态增强**：
  1. 自定义 TabBar 容器增加顶部细分割线，提升导航区域边界感。
  2. 激活态增加字重与图标轻微放大位移，形成“颜色 + 形态”双重反馈。
  3. 同步拉开激活色与未激活色对比，确保当前页可快速识别。

## 「我的」页新增交互调整（本次变更）

- **顶部文案动态化**：将固定文案「个人中心」替换为动态展示，优先取登录态中的 `nickname`，若为空则回退到 `name`，再回退到默认文案「我的」。
- **退出登录入口上移**：将退出登录按钮从 `subpkg/mine/setting/index` 移至 `pages/mine/index`，作为一级页面底部操作区；保留原确认弹窗逻辑与 `auth/LogOut` 调用，不改动登出流程。
- **布局约束**：退出按钮与菜单区之间保持明显间距，并遵循现有主题变量（错误色、边框色）保证深浅色模式一致性。

## 「我的」页菜单层级与顺序调整（本次变更）

- **去除应用设置聚合入口**：一级菜单不再展示「应用设置」，避免用户多一级跳转。
- **一级菜单顺序**（自上而下）：
  1. 编辑资料
  2. 修改密码
  3. 深色模式（开关）
  4. 关于我们
  5. 退出登录
- **应用设置页收敛**：`subpkg/mine/setting/index` 中移除「检查更新」「清理缓存」两项；若页面保留，仅保留与主题相关能力，不再作为一级常用入口。
- **交互约束**：
  1. 「修改密码」从我的页一级直达 `subpkg/mine/pwd/index`。
  2. 「深色模式」在我的页一级直接切换，不增加二次确认。
  3. 「退出登录」继续复用原确认弹窗与 `auth/LogOut` 流程。
- **图标规范**：`修改密码`、`深色模式`、`关于我们` 三个菜单图标统一改为 Vant Icon，分别使用 `lock`、`setting-o`、`info-o`，避免旧 iconfont 图标语义弱和风格不一致。

## 「编辑资料」页头像更新能力扩展（本次变更）

- **入口位置**：在 `subpkg/mine/info/edit` 的表单顶部新增「头像」项，展示当前头像缩略图。
- **交互行为**：点击头像项后跳转 `subpkg/mine/avatar/index`，复用既有裁剪与上传提交流程 `postCurrentUserUploadAvatar`，不新增重复上传实现。
- **数据一致性**：头像上传成功后依赖现有 `store` 中 `auth.avatar` 更新机制，返回编辑页或我的页后应立即看到新头像。

## 「修改密码 / 关于我们」深色模式与密码安全校验（本次变更）

- **导航栏主题一致性**：
  1. `subpkg/mine/pwd/index` 与 `subpkg/mine/about/index` 的 `uni-nav-bar` 不再写死浅色值。
  2. 导航栏背景与文字颜色统一绑定主题变量（`--t-nav-bg`、`--t-nav-text`），保证深浅模式切换时视觉一致。
- **修改密码表单增强**：
  1. 表单字段从两项扩展为三项：`old_password`（当前密码）、`password`（新密码）、`password_two`（确认密码）。
  2. 前端提交前保持已有新密码长度/两次一致性校验，并新增当前密码必填校验。
- **后端校验增强**：
  1. `ResetPwd` 入参模型新增 `old_password` 必填字段。
  2. 重置密码前先校验当前密码与数据库哈希一致；不一致直接返回业务错误，不更新密码。
  3. 当前密码校验通过后，再沿用既有新密码复杂度与双录一致性校验流程。

## TabBar 真机尺寸协调优化（本次变更）

- **问题现象**：部分机型上自定义 TabBar 视觉尺寸偏大，图标与文字占比过重，影响页面整体协调性。
- **设计调整**：
  1. `custom-tab-bar/index.wxss` 中将 `--tabbar-height` 从 `56px` 调整为 `50px`，降低底栏占高。
  2. `--tabbar-item-font-size` 从 `16px` 调整为 `13px`，`--tabbar-item-icon-size` 从 `28px` 调整为 `24px`，并将图文间距下调至 `3px`。
  3. 同步更新 `custom-tab-bar/theme.js` 与 `uni.scss` 的 TabBar 设计令牌（`$mp-tabbar-height`、`$mp-tabbar-label-px`、`$mp-tabbar-icon-px`），确保样式与配置一致，避免后续漂移。
- **兼容性约束**：保持 `safe-area-inset-bottom` 开启，不改变点击命中逻辑与路由切换行为，仅做视觉密度优化。

## TabBar 激活态真机同步稳定性优化（本次变更）

- **问题现象**：真机上偶发出现「页面已切换，但底部激活态仍停留在旧 Tab」；模拟器不稳定复现。
- **设计调整**：
  1. `custom-tab-bar/index.js` 的激活态同步由「立即 + 80ms 二次同步」升级为「多次短间隔重试同步」（`0/80/180/320ms`），覆盖微信真机路由栈延迟更新窗口。
  2. 在 `switchTab` 触发时先更新组件本地 `activePath`，确保交互反馈即时；切换完成后再次触发重试同步，保证最终态以当前页路由为准。
  3. 切换失败时同样执行兜底同步，避免激活态停留在错误值。
- **约束**：Tab 激活态最终值必须由当前页面路由校准，不允许仅依赖点击事件一次性赋值。

## TabBar 激活态路由事件优先校准（本次变更）

- **问题现象**：在部分真机上，页面已切换到「商业」等目标页，但底部激活态仍停留在旧页（如「需求」）。
- **设计调整**：
  1. 在 `custom-tab-bar/index.js` 中新增 `wx.onAppRoute` 监听，路由变化后优先使用事件路径直接校准 `activePath`，避免页面栈更新滞后导致误判。
  2. 将激活态重试窗口扩展为 `0/80/180/320/500/800/1200ms`，在慢机型上继续以页面栈作为兜底校准来源。
  3. 组件销毁时调用 `wx.offAppRoute` 解绑监听，避免重复注册造成状态抖动。
- **约束**：激活态同步优先级固定为「路由事件路径 > 页面栈路径 > 点击瞬时赋值」。

## 约束

- 分包 `subpkg/mine` 下各子页（个人信息、常见问题详情等）不因本需求删除文件；仅主列表不再跳转「个人信息」「常见问题」。
- 登录态仍由全局与路由 meta 控制；本页不再提供从顶部进入「个人信息」的路径，用户仍可通过「编辑资料」进入资料相关子页（与 OR「保留编辑资料」一致）。
- 真机环境下 `switchTab` 后的激活态校准需要至少一次延迟重试；仅靠 `pageLifetimes.show` 单次同步不作为充分条件。
