# 功能设计

## 小程序登录入口

- 登录页提供两种入口：`账号密码登录` 和 `微信授权登录`。
- 账号密码登录输入框字段名沿用 `telephone`（兼容历史客户端），语义为「手机号或账号」。
- 密码输入默认掩码显示，支持显隐切换（仅前端展示态切换，不改变值）。

## 账号密码登录（`POST /auth/login`）

- 请求保持现有结构：`telephone` + `password` + `method=0` + `platform=1`。
- 登录标识解析规则保持现有实现：
  - 合法手机号：按 `telephone` 精确匹配；
  - 含字母账号：先 `name` 精确匹配，再 `nickname` 精确匹配。
- 失败文案继续使用统一口径：
  - 账号不存在/密码错误：`账号或密码错误`；
  - 冻结：`此账号已被冻结`；
  - 无权限：`此账号无权限`。

## 微信授权登录（`POST /auth/wx/login`）

- 新增入参：`nickname`、`avatar`（前端通过 `wx.getUserProfile` 获取）。
- `code` 继续用于服务端换取手机号；手机号获取失败时返回 `无效Code`。
- 微信配置读取优先级：`.env` 中 `WECHAT_APPID` + `WECHAT_KEY` > 系统设置表 `wx_server_app_id` + `wx_server_app_secret`（用于兼容历史配置）。
- 服务端处理流程：
  1. 通过 `code` 解析手机号；
  2. 根据手机号查询用户；
  3. 若不存在则自动创建微信用户；
  4. 若已存在则更新用户微信档案字段（昵称/头像）并刷新用户类型；
  5. 生成 token 并返回登录结果。

## 微信用户与系统用户区分

- 在 `vadmin_auth_user` 增加字段：
  - `user_type`：用户类型（`system` / `wechat`）；
  - `wx_nickname`：微信昵称；
  - `wx_avatar`：微信头像地址。
- 字段语义：
  - `system`：通过系统创建或账号密码链路登录为主的用户；
  - `wechat`：通过小程序微信授权创建或更新的用户。
- 自动创建微信用户时默认赋值：
  - `user_type=wechat`；
  - `name` 由 `nickname` 兜底生成（无昵称时使用手机号后 4 位）；
  - `nickname/wx_nickname`、`avatar/wx_avatar` 用授权信息填充；
  - `is_staff=True`、`is_active=True`，并生成随机初始密码哈希以兼容现有 token 校验流程。

## 登录响应扩展

- `/auth/login` 与 `/auth/wx/login` 返回体新增 `user_type` 字段，供前端识别用户来源。
- 兼容保留：`access_token`、`refresh_token`、`token_type`、`is_reset_password`、`is_wx_server_openid`。

# 数据模型

- 表：`vadmin_auth_user`
- 新增字段：
  - `user_type` `String(16)` 非空，默认 `system`；
  - `wx_nickname` `String(128)` 可空；
  - `wx_avatar` `String(500)` 可空。
- 迁移方式：通过 Alembic 增量迁移新增字段；历史用户回填 `user_type=system`。

# 约束与边界

- 微信授权链路依赖微信平台能力（手机号授权、用户资料授权）；任一授权失败时应允许用户回退到账号密码登录。
- 同手机号重复授权不重复建号，只更新微信档案并复用原账号。
- 现阶段不引入数据库外键或新关联表，用户类型信息直接存储在用户表字段中。
- 用户类型仅用于来源区分与业务判定，不影响现有角色权限模型。

## 启动初始化补充（`hipp-api/run.sh`）

- 启动前继续通过核心表存在性检查决定是否执行 `init`。
- 当目标数据库连接返回 MySQL `1049 (Unknown database)` 时，脚本应自动执行建库：
  - 使用相同主机、端口、用户名、密码建立不指定库名的连接；
  - 执行 `CREATE DATABASE IF NOT EXISTS <db_name> CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci`。
- 建库成功后重连目标库并恢复原流程：
  - 核心表存在：直接启动服务；
  - 核心表缺失：执行迁移和种子初始化后启动服务。
- 仅数据库不存在场景启用自动建库；权限不足、网络不可达、认证失败等异常仍保持失败退出。

# 前端（hipp-uni）

- 登录页新增微信授权登录按钮与授权说明（底部弹层展示头像/昵称/手机号授权项）。
- 微信登录前先获取用户资料，再处理手机号授权码并调用后端接口。
- 前端状态管理新增 `userType` 持久化字段，用于区分微信用户和系统用户展示/后续分流。
