# 功能设计

## 入口

- 管理端 **系统管理 > 系统配置**（`hipp-admin` `Settings.vue`）。
- 在动态加载的站点配置 Tab 之后，**追加固定 Tab**「数据导入导出」，组件 `DataImportExport.vue`。

## 数据包格式

- **ZIP**，根目录含 `manifest.json`，`data/` 下每个业务表一个 `表名.json`（JSON 数组，元素为行对象）。
- `manifest.json` 字段：
  - `format_version`：当前为 `1.0`
  - `app`：固定 `hipp-or`
  - `exported_at`、`exported_by`
  - `import_strategy`：`full_replace`（全量替换）
  - `tables`：表名字符串数组，须与后端 `TABLES_ORDER` 集合一致

## 覆盖表清单（与代码 `TABLES_ORDER` 一致）

包含权限与组织、字典、系统配置、智能体、资源、帮助中心、小程序会话与消息、定时任务与 APScheduler 作业表、操作/登录/短信记录等共 **25** 张表（不含 Alembic 版本表等迁移元数据）。具体表名以实现代码为准。

## 导入策略（默认）

- **全量替换**：单事务内 `SET FOREIGN_KEY_CHECKS=0`，按逆序 `DELETE` 各表，再按正序逐行 `INSERT` 恢复导出包中的主键与数据。
- 校验：ZIP、`manifest` 版本与 `app`、表集合与当前系统一致；否则拒绝导入且不修改数据。

## API

| 方法 | 路径 | 鉴权 | 说明 |
|:---|:---|:---|:---|
| POST | `/vadmin/system/data-migration/export` | FullAdminAuth | 返回 `application/zip` 附件 |
| POST | `/vadmin/system/data-migration/import` | FullAdminAuth | `multipart/form-data`，字段名 `file`，最大 100MB |

## 约束与边界

- 演示环境（`HIPP_DEMO`）下已将上述路径加入白名单，否则 POST 会被拦截。
- 导入为高危操作，界面需显著提示；密码等敏感字段随业务表一并导出，需管理端权限与可信网络。
